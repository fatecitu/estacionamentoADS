# Projeto Estacionamento

Sistema web simples para controle de estacionamento, desenvolvido com Vite, Tailwind CSS e Supabase.

## Funcionalidades

- Login de usuarios com Supabase Auth.
- Cadastro de novos usuarios.
- Menu principal do sistema.
- Cadastro, listagem, edicao e exclusao de marcas e modelos de veiculos.
- Logout da sessao autenticada.

## Tecnologias

- HTML
- JavaScript
- Vite
- Tailwind CSS
- Supabase

## Requisitos

- Node.js instalado.
- Conta e projeto criados no Supabase.

## Como executar

Instale as dependencias:

```bash
npm install
```

Crie um arquivo `.env` na raiz do projeto com as chaves do Supabase:

```env
VITE_SUPABASE_URL=sua-url-do-supabase
VITE_SUPABASE_PUBLISHABLE_KEY=sua-chave-publica-do-supabase
```

Inicie o servidor de desenvolvimento:

```bash
npm run dev
```

Acesse a URL exibida no terminal, normalmente:

```text
http://localhost:5173
```

## Configuracao do Supabase

O projeto usa a autenticacao do Supabase para login e cadastro. Ative o provedor de e-mail/senha no painel do Supabase.

Para a tela de modelos funcionar, crie uma tabela chamada `modelos` com os campos:

| Campo | Tipo | Observacao |
| --- | --- | --- |
| `id` | `int8` ou `uuid` | Chave primaria |
| `marca` | `text` | Marca do veiculo |
| `descricao` | `text` | Modelo do veiculo |

Exemplo de SQL:

```sql
create table modelos (
  id bigint generated by default as identity primary key,
  marca text not null,
  descricao text not null
);
```

Se estiver usando Row Level Security, configure politicas de acesso para permitir as operacoes usadas pelo sistema.

## Estrutura principal

```text
.
+-- index.html          # Tela de login
+-- menu.html           # Menu principal
+-- modelos.html        # CRUD de marcas e modelos
+-- novo.html           # Cadastro de usuario
+-- src/
|   +-- main.js         # Logica de login
|   +-- menu.js         # Logica do menu e logout
|   +-- modelos.js      # CRUD de modelos
|   +-- novo.js         # Cadastro de usuario
|   +-- style.css       # Estilos com Tailwind
|   +-- supabase.js     # Cliente Supabase
+-- vite.config.js      # Configuracao do build multipagina
```

## Observacoes

- As variaveis de ambiente devem comecar com `VITE_` para serem lidas pelo Vite.
- O arquivo `.env` nao deve ser enviado para repositorios publicos.
- O arquivo `src/.env-example` mostra o formato esperado das variaveis.
